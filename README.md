# Шаблон репозитория для практикума по контейнерам

Этот репозиторий — стартовый шаблон для студенческой группы.

## Что уже настроено

- CMake-проект
- Makefile для локальной сборки
- генерация проекта Visual Studio 2022
- GitHub Actions для CI
- пример unit-теста на Boost.Test

## Быстрый старт

### Linux / WSL

```bash
make build
make test
```

### Visual Studio 2022

```bash
make vs
make vs-build
make vs-test
```

После `make vs` можно открыть solution-файл из папки `build-vs`.

## Рекомендуемый процесс работы

- `main` — стабильная ветка
- `develop` — общая ветка разработки группы
- `feature/...` — рабочие ветки студентов

Изменения вносятся через Pull Request с обязательным code review и зелёным CI.

## Что должна сделать группа

- добавить реализации своих контейнеров
- расширить набор unit-тестов
- договориться о правилах code review
- не коммитить напрямую в `main`
